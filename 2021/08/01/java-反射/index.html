java-反射

发表于 2021-08-01 分类于 java
反射是个很强大的功能，这里只是简单地介绍下基础应用。

1. 什么是反射

之前也百度过这个问题，但都是比较冗长的概念。不过，《java核心技术 卷1》里面的解释我觉得很好，描述如下：

能够分析类能力的程序称为反射。

我之所以觉得这句解释比较好是因为它简单明了，在学习过程中，我们用到的99%都是实例对象，可是，实例对象都是先有类，再有对象，反射的关注点不在对象，而在类！因此，这种关注点的转换，让我觉得很好地体现了”反“这个字。当然，除此之外，也说明了它的用途：分析类能力，以便操作。

2. 反射的用途

上面说，反射可以分析类能力，那么分析类能力到底是为了什么？如下：

  1. 在运行时分析类能力
  2. 在运行时查看对象，例如，编写一个toString方法供所有类使用
  3. 实现通用的数组操作代码
  4. 利用Method对象，实现”指针式“的万能操作（这才是重点）

4. 利用反射分析类的能力

在java.lang.reflect 包中有三个类 Field、Method 和 Constructor 分别用于描述类的域、方法和构造器。这三个类都有一个叫做 getName 的方法，用来返回项目的名称。

Field 类有一个 getType 方法，用来返回描述域所属类型的Class对象。Method 和 Constructor 类有能够报告参数类型的方法，Method 类还有一个可以报告 返回类型 的方法。

这三个类还有一个叫做 getModifiers 的方法，它将返回一个整型数值，用不同的位开关描述public 和 static 这样的修饰符使用状况。另外，还可以利用 java.lang.reflect 包中的Modifier类的静态方法分析 getModifiers 返回的整型数值。例如，可以使用 Modifier 类中的isPublic、isPrivate 或isFinal判断方法或构造器是否是public、private 或final。

5. 调用私有属性和方法

注意！

Class类中的 getFields、``getMethods、getConstructors方法只能返回该类提供的public域、方法和构造器数组，其中**包括父类**的public` 成员。

如果想调用该类的私有成员和方法，对应的应该调用 getDeclaredField、``getDeclaredField、getDeclaredMethod` 方法。当然，并不是说这几个方法只能返回私有属性的，事实上，他们可以返回该类中声明的全部属性、方法和构造器，但是，其中并不包括父类的成员。也就是说，只能访问自己的。
